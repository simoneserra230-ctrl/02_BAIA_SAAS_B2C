AI Analisi Bandi v2.1

Piattaforma SaaS per l'analisi automatica di bandi di finanziamento italiani ed europei tramite AI (Groq + llama-3.3-70b-versatile). Supporta PDF fino a ~350 pagine, analisi strutturata in 18 sezioni, matching aziende/bandi e generazione schede complete.

🧠 Motore AI

Groq · llama-3.3-70b-versatile
128k token · alta precisione

🔌 Backend

FastAPI + Python 3.10+
Porta 8000 (locale)

🖥 Frontend

HTML singolo file · nessun build
Aprire direttamente nel browser

☁️ Cloud (opzionale)

Supabase (DB + Auth) · Stripe (pagamenti)
Non richiesti per uso locale

Installazione

1 · Estrarre i file

Estrarre il contenuto dello ZIP in una cartella a scelta, ad esempio C:\BAIA\ oppure ~/baia/. La struttura risultante deve essere:

02_BAIA_SAAS_B2C/
├── main.py                    # ← backend FastAPI
├── AI_Analisi_Bandi_SaaS.html # ← frontend (aprire nel browser)
├── requirements.txt
├── .env.example
├── avvia.bat                  # ← Windows: doppio clic per avviare
└── venv/                      # ← ambiente virtuale Python

2 · Creare l'ambiente virtuale (consigliato)

Aprire un terminale nella cartella del progetto ed eseguire:

# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate

3 · Installare le dipendenze

pip install -r requirements.txt
✅
Se usi avvia.bat su Windows, le dipendenze vengono installate automaticamente ad ogni avvio.

Configurazione .env

Creare il file .env

# Windows (terminale o Esplora file)
copy .env.example .env

# macOS / Linux
cp .env.example .env

Aprire il file .env con un editor di testo (Notepad, VS Code, ecc.) e compilare i valori:

# OBBLIGATORIO: inserire la propria chiave Groq
GROQ_API_KEY=gsk_xxxxxxxxxxxxxxxxxxxxxxxxxxxx

# Modello AI — lasciare invariato per risultati ottimali
GROQ_MODEL=llama-3.3-70b-versatile

# Supabase e Stripe: lasciare i valori placeholder
# se non si usa la modalità SaaS cloud
⚠️
Non condividere mai il file .env. Non caricarlo su GitHub o servizi cloud. Contiene credenziali private.

Ottenere la chiave API Groq

  • 1

    Aprire la console Groq

    Visitare console.groq.com/keys e creare un account gratuito (accesso con Google o email).

  • 2

    Creare una nuova API Key

    Cliccare su Create API Key, dare un nome (es. baia-locale) e copiare la chiave generata. Inizia sempre con gsk_.

  • 3

    Incollare nel file .env

    Aprire .env e sostituire gsk_LA_TUA_CHIAVE_GROQ con la chiave copiata. Salvare il file.

ℹ️
Piano gratuito Groq: include generosi limiti mensili (decine di migliaia di token al giorno). Sufficiente per un utilizzo professionale normale. I limiti vengono gestiti automaticamente dal backend con retry.

Avviare il backend

Windows — metodo rapido

Fare doppio clic su avvia.bat. Il file verifica automaticamente la configurazione, installa le dipendenze e avvia il server.

Terminale (Windows / macOS / Linux)

# Dalla cartella del progetto, con venv attivo
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

Output atteso all'avvio corretto:

[STARTUP] .env caricato correttamente
[STARTUP] Groq configurato ✓  modello=llama-3.3-70b-versatile
[STARTUP] Modalità: locale (no Supabase)
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Application startup complete.
🔴
Se vedi [WARNING] GROQ_API_KEY non impostata: il file .env non esiste, non è nella cartella giusta, oppure la chiave non è stata inserita correttamente.

Aprire il frontend

Con il backend in esecuzione, aprire il file AI_Analisi_Bandi_SaaS.html direttamente nel browser (doppio clic o drag & drop nella barra indirizzi). Non è necessario un server web per il frontend.

ℹ️
All'apertura appare una schermata di login. Per uso locale, cliccare "🎭 Prova la Demo senza registrarti" per entrare direttamente. Nessun account necessario in LOCAL_MODE.

Verifica connessione frontend ↔ backend

Dopo aver aperto il frontend, verificare che sia collegato al backend:

  • 1

    Indicatore modello AI

    In alto a destra nella piattaforma compare il nome del modello attivo (es. llama-3.3-70b-versatile). Se mostra "N/D" o "backend non raggiungibile", il backend non è avviato o è sulla porta sbagliata.

  • 2

    Test API diretto

    Aprire nel browser: http://localhost:8000 — deve rispondere con un JSON che include "status": "ok" e il modello configurato.

  • 3

    Documentazione API automatica

    Disponibile su http://localhost:8000/docs — elenco completo di tutti gli endpoint con possibilità di test interattivo.

Analizzare un bando

  • 1

    Caricare il PDF del bando

    Nella sezione Bandi, cliccare "+ Carica Bando" e selezionare uno o più file PDF. Sono supportati bandi multi-file (es. bando principale + allegati tecnici).

  • 2

    Selezionare la modalità di analisi

    Scegliere tra Analisi Rapida (più veloce, ~30 secondi) e Analisi Accurata (più completa, analizza ogni documento separatamente e poi fonde i risultati). Per bandi importanti usare sempre la modalità Accurata.

  • 3

    Avviare l'analisi

    Cliccare "Analizza". Il backend estrae il testo dal PDF, lo invia a Groq e riceve la scheda strutturata. Per PDF grandi (>100 pagine) attendere 60–120 secondi.

  • 4

    Revisionare la scheda

    Il risultato viene mostrato nella scheda bando con 18 sezioni strutturate. I campi non trovati nel PDF vengono evidenziati e possono essere arricchiti con la funzione "Arricchisci campi".

Modalità di analisi

ModalitàLimite testo/fileQuando usarlaTempo stimato
⚡ Rapida 80.000 caratteri (~40 pag.) Bandi brevi, pre-screening veloce 15–40 sec
🎯 Accurata 120.000 caratteri/file (~60 pag.) Bandi complessi, analisi definitiva 60–150 sec
ℹ️
Bandi molto lunghi (>80 pagine): il backend divide automaticamente il testo in blocchi da 60.000 caratteri, analizza ogni blocco e unisce i risultati. Questo avviene in modo trasparente.

Profili aziendali

Nella sezione Aziende è possibile creare e gestire profili aziendali dettagliati. Ogni profilo include: settore ATECO, dimensione, fatturato, regione, forma giuridica, anni di attività e altri parametri rilevanti per il matching con i bandi.

Creare un profilo aziendale

  • 1

    Cliccare "+ Nuova Azienda"

    Compilare i campi del form. I campi obbligatori sono: nome, settore, dimensione (micro/piccola/media/grande), regione.

  • 2

    Inserire i dati finanziari

    Fatturato, numero dipendenti e totale attivo permettono un matching più preciso con i requisiti economici dei bandi.

Matching bando / azienda

Il sistema calcola automaticamente la compatibilità tra ogni bando analizzato e i profili aziendali salvati. Il punteggio di matching considera: settore, dimensione, regione, requisiti finanziari e forma giuridica.

Per avviare il matching: aprire la scheda di un bando e cliccare "Verifica Compatibilità". Il risultato mostra un punteggio (0–100) e i criteri soddisfatti/non soddisfatti per ogni azienda.

Modalità SaaS (Supabase + Stripe)

Per attivare la sincronizzazione cloud e la gestione abbonamenti, configurare le variabili aggiuntive nel file .env e cambiare la modalità nel frontend:

# In AI_Analisi_Bandi_SaaS.html, circa riga 2275
const SAAS_CONFIG = {
  supabaseUrl:     'https://TUA_URL.supabase.co',
  supabaseAnonKey: 'eyJ...',
  backendUrl:      'http://localhost:8000',
  mode:            'saas',  // ← cambia da 'local' a 'saas'
};
⚠️
In modalità LOCAL (default) tutti i dati sono salvati solo nel browser (localStorage). Non c'è sincronizzazione tra dispositivi e i dati si perdono pulendo il browser. La modalità SaaS richiede un account Supabase.

Risoluzione problemi comuni

Backend non si avvia

ErroreSoluzione
ModuleNotFoundError: fastapi Eseguire pip install -r requirements.txt con l'ambiente virtuale attivo
Address already in use :8000 Un'altra istanza del backend è già in esecuzione. Chiuderla o cambiare porta: --port 8001
python: command not found Installare Python da python.org e riavviare il terminale

Frontend non si connette al backend

SintomoSoluzione
Modello AI mostra "N/D" Verificare che il backend sia in esecuzione su localhost:8000
Errore CORS in console browser Il backend è su una porta diversa da 8000? Modificare backendUrl in SAAS_CONFIG nel HTML
Analisi non parte, nessun errore Aprire DevTools (F12) → Console e verificare il messaggio di errore esatto

Errori durante l'analisi AI

ErroreSoluzione
401 Unauthorized da Groq La GROQ_API_KEY è errata o scaduta. Verificare su console.groq.com
model not found Il modello in GROQ_MODEL non esiste. Usare llama-3.3-70b-versatile
Rate limit frequenti Il backend gestisce automaticamente i retry. Per uso intensivo considerare il piano a pagamento Groq
Analisi tronca / JSON incompleto Il PDF ha testo non estrabile (scansione immagini). Usare un PDF con testo selezionabile
Solo file PDF supportati Il file caricato non è un PDF. Convertire con Word, LibreOffice o un tool online

Modifiche apportate nella versione 2.1

✅
Tutti i bug critici che impedivano il funzionamento sono stati risolti.
Problema (v2.0)Fix applicato (v2.1)
File .env non veniva mai caricato → GROQ_API_KEY sempre vuota → tutte le analisi fallivano con errore 401 Aggiunto load_dotenv(override=True) all'inizio di main.py, prima di qualsiasi os.environ.get()
Modello meta-llama/llama-4-scout-17b-16e-instruct non esiste su Groq → errore "model not found" ad ogni richiesta Sostituito con llama-3.3-70b-versatile (modello stabile, 128k token, massima precisione)
Limite testo frontend 14.000–28.000 caratteri → i bandi venivano troncati all'~8% del contenuto reale Nuovi limiti: 80.000 (rapida) e 120.000 (accurata) caratteri per file. Bandi fino a ~60 pagine analizzati integralmente
max_tokens: 4096 in Groq → risposte JSON troncate su bandi complessi → parse falliva Portato a max_tokens: 8192 per garantire JSON completo anche per schede con tutte le 18 sezioni
CHUNK_SIZE = 30_000 e SIMPLE_LIMIT = 80_000 inadeguati per PDF lunghi Nuovi valori: CHUNK_SIZE = 60_000, SIMPLE_LIMIT = 320_000 (~80k token, nel limite 128k del modello)
avvia.bat non verificava la presenza del .env né della chiave Groq Nuovo avvia.bat con controlli espliciti: verifica .env, chiave Groq, Python, dipendenze
